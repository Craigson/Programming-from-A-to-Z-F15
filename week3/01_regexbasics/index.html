Regex Testing

The function test() returns true or false if the regex finds a match:

var text = 'This is a test of regular expressions.'; var regex = /test/; var results = regex.test(text); console.log(results);

The function search() returns the index of a match:

var text = 'This is a test of regular expressions.'; var regex = /test/; var results = text.search(regex); console.log(results);

The string function match() returns all the data around a match, the match, any captured groups, the index of the match, and input text.

var text = 'This is a test of regular expressions.'; var regex = /test (of)/; var results = text.match(regex); console.log(results);

The function exec() does exactly the same thing, returning all the data around a match.

var text = 'This is a test of regular expressions.'; var regex = /test/; var results = regex.exec(text); console.log(results);

The difference between match() and exec() becomes apparent when using the global flag. match() returns an array of all the matches, but does not include the captured groups.

var text = 'Now another test including phone numbers: 212-555-1234 and 917-555-4321 and 646.555.9876.'; var regex = /(\d+)[-.]\d+[-.]\d+/g; var results = text.match(regex); console.log(results);

If you want to get all of the captured groups with the global flag, you can use exec() in a loop. exec() will return all the data around each match each time it is called and null when no more matches found.

var text = 'Now another test including phone numbers: 212-555-1234 and 917-555-4321 and 646.555.9876.'; var regex = /(\d+)[-.]\d+[-.]\d+/g; var results = regex.exec(text); while (results !== null) { console.log(results); results = regex.exec(text); }
